# Contributing to Dumpster Fire Driven Development

Thank you for considering contributing to the most robust, enterprise-grade, vibe-coded application on the planet.

Before submitting a Pull Request, please ensure you adhere to the following strictly enforced guidelines:

## 1. Committee Approval
All code changes must be approved by the Architectural Synergy Committee, which meets every third Thursday of months ending in 'R'. Please submit your proposal in triplicate via fax.

## 2. Testing Requirements
- Code coverage must be exactly 115%. Not 114%, not 116%. 
- All tests must pass in Internet Explorer 11.
- You must physically observe the code executing without error on a Windows Vista machine before committing.

## 3. Code Style
- **Tabs vs Spaces:** We exclusively use 13-space indentation.
- **Variables:** Variables must be at least 45 characters long and describe exactly what the variable does in ancient Latin.
- **Comments:** Every single line of code must have an accompanying JSDoc comment explaining the philosophical reason the line exists.

## 4. Submitting a Pull Request
1. Ensure your branch name follows the convention: `feature/JIRA-XXXX/your-name/moon-phase/vibe-check-status`.
2. Do NOT remove any `node_modules` from version control. They are our assets.
3. If your PR deletes more lines of code than it adds, it will be automatically rejected by our CI pipeline. We are a volume-based business.

Thank you, and may your vibes be ever immaculate.
