# Dumpster Fire Driven Development (DFDD) 🔥

Welcome to the future of software engineering. 

While other developers are trying to write *less* code and reduce their "technical debt" (a concept we believe is a myth invented by HR to pay you less), we are embracing **Dumpster Fire Driven Development (DFDD)**. 

Our core philosophy: **Code is an Asset. More Code = More Assets.** 

If you aren't writing 300,000 Lines of Code (LOC) a week, are you even an engineer?

## Core Tenets of DFDD
1. **Never use a built-in method when a 500-line custom implementation will do.**
2. **If it doesn't have 14 layers of abstraction, it's not Enterprise-Ready.**
3. **`node_modules` is an asset class. Commit it to version control.**
4. **User Experience (UX) builds character. Make the user work for it.**

## The Project
This repository is a shining example of DFDD. It contains the world's most advanced, highly-available, web-scale, synergized To-Do application (or something similar), powered by millions of lines of meticulously generated, highly repetitive, artisanal code.
